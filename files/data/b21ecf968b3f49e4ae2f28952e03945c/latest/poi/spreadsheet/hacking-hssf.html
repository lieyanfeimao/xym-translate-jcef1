Hacking HSSF(黑客 HSSF)

Where Can I Find Documentation on Feature X(我在哪里可以找到有关功能 X 的文档)

You might find the 'Excel 97 Developer's Kit' (out of print, Microsoft Press, no restrictive covenants, available on Amazon.com) helpful for understanding the file format.(您可能会发现“Excel 97 Developer's Kit”（绝版，Microsoft Press，无限制性契约，可在 Amazon.com 上获得）有助于理解文件格式。)

Also useful is the open office XLS spec. We are collaborating with the maintainer of the spec so if you think you can add something to their document just send through your changes.(开放式办公室 XLS 规范也很有用。我们正在与规范的维护者合作，因此如果您认为可以在他们的文档中添加一些内容，只需发送您的更改即可。)

Help, I Can't Find Feature X Documented Anywhere(帮助，我在任何地方都找不到功能 X)

  1. Look at OpenOffice.org or Gnumeric sources if its implemented there.(查看 OpenOffice.org 或 Gnumeric 源（如果在那里实现）。)
  2. Use org.apache.poi.hssf.dev.BiffViewer to view the structure of the file. Experiment by adding one criteria entry at a time. See what it does to the structure, infer behavior and structure from it. Using the unix diff command (or get cygwin from www.cygwin.com for windows) you can figure out a lot very quickly. Unimplemented records show up as 'UNKNOWN' and prints a hex dump.(使用 org.apache.poi.hssf.dev.BiffViewer 查看文件的结构。通过一次添加一个条件条目进行实验。查看它对结构的作用，从中推断行为和结构。使用 unix diff 命令（或从 www.cygwin.com 获取 windows 的 cygwin），您可以非常快速地计算出很多东西。未实现的记录显示为“未知”并打印一个十六进制转储。)

Low-level Record Generation(低级记录生成)

Low level records can be time consuming to created. We created a record generator to help generate some of the simpler tasks.(创建低级别记录可能很耗时。我们创建了一个记录生成器来帮助生成一些更简单的任务。)

We use XML descriptors to generate the Java code (which sure beats the heck out of the PERL scripts originally used ;-) for low level records. The generator is kinda alpha-ish right now and could use some enhancement, so you may find that to be about 1/2 of the work. Notice this is in org.apache.poi.hssf.record.definitions.(我们使用 XML 描述符来为低级记录生成 Java 代码（这肯定优于最初使用的 PERL 脚本 ;-)。生成器现在有点 alpha-ish 并且可以使用一些增强功能，所以您可能会发现这大约是工作的 1/2。请注意，这是在 org.apache.poi.hssf.record.definitions 中。)

Important Notice(重要通知)

One thing to note: If you are making a large code contribution we need to ensure any participants in this process have never signed a "Non Disclosure Agreement" with Microsoft, and have not received any information covered by such an agreement. If they have they'll not be able to participate in the POI project. For large contributions we may ask you to sign an agreement.(需要注意的一点：如果您要进行大量代码贡献，我们需要确保此过程中的任何参与者从未与 Microsoft 签署过“保密协议”，并且未收到此类协议所涵盖的任何信息。如果他们有，他们将无法参与 POI 项目。对于大额捐款，我们可能会要求您签署协议。)
