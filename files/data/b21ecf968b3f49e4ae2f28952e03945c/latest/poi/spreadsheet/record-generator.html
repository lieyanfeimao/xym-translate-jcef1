Record Generator HOWTO(记录生成器 HOWTO)

How to Use the Record Generator(如何使用记录生成器)

History(历史)

The record generator was born from frustration with translating the Excel records to Java classes. Doing this manually is a time consuming process. It's also very easy to make mistakes.(记录生成器是在将 Excel 记录转换为 Java 类的过程中产生的。手动执行此操作是一个耗时的过程。也很容易出错。)

A utility was needed to take the definition of what a record looked like and do all the boring and repetitive work.(需要一个实用程序来定义记录的样子并完成所有无聊和重复的工作。)

Capabilities(能力)

The record generator takes XML as input and produces the following output:(记录生成器将 XML 作为输入并产生以下输出：)

  • A Java file capable of decoding and encoding the record.(能够对记录进行解码和编码的 Java 文件。)
  • A test class that provides a fill-in-the-blanks implementation of a test case for ensuring the record operates as designed.(一个测试类，它提供测试用例的填空实现，以确保记录按设计运行。)

Usage(用法)

The record generator is invoked as an Ant target (generate-records). It goes through looking for all files in src/records/definitions ending with _record.xml. It then creates two files; the Java record definition and the Java test case template.(记录生成器作为 Ant 目标调用（生成记录）。它会在 src/records/definitions 中查找以 _record.xml 结尾的所有文件。然后它创建两个文件； Java 记录定义和 Java 测试用例模板。)

The records themselves have the following general layout:(记录本身具有以下总体布局：)

record id="0x1032" name="Frame" package="org.apache.poi.hssf.record"
excel-record-id="FRAME">
<description>The frame record indicates whether there is a border
around the displayed text of a chart.</description>
<author>Glen Stampoultzis (glens at apache.org)</author>
<fields>
<field type="int" size="2" name="border type">
<const name="regular" value="0" description="regular rectangle or no border"/>
<const name="shadow" value="1" description="rectangle with shadow"/>
</field>
<field type="int" size="2" name="options">
<bit number="0" name="auto size"
description="excel calculates the size automatically if true"/>
<bit number="1" name="auto position"
description="excel calculates the position automatically"/>
</field>
</fields>
/record>

The following table details the allowable types and sizes for the fields.(下表详细说明了字段的允许类型和大小。)

Type(类型) Size(尺寸) Java Type(Java 类型)
int(整数) 1(1) byte(字节)
int(整数) 2(2) short(短的)
int(整数) 4(4) int(整数)
int(整数) 8(8) long(长)
int(整数) varword(变量字) array of shorts(短裤阵列)
bits(位) 1(1) A byte comprising of a bits (defined by the bit element)(由位组成的字节（由位元素定义）)
bits(位) 2(2) An short comprising of a bits(一个由位组成的短)
bits(位) 4(4) A int comprising of a bits(由位组成的 int)
float(漂浮) 8(8) double(双倍的)
hbstring(hbstring) java expression(java表达式) String(细绳)

The Java records are regenerated each time the record generator is run, however the test stubs are only created if the test stub does not already exist. What this means is that you may change test stubs but not the generated records.(每次运行记录生成器时都会重新生成 Java 记录，但是只有在测试存根不存在时才会创建测试存根。这意味着您可以更改测试存根，但不能更改生成的记录。)

Custom Field Types(自定义字段类型)

Occasionally the builtin types are not enough. More control over the encoding and decoding of the streams is required. This can be achieved using a custom type.(有时内置类型是不够的。需要对流的编码和解码进行更多控制。这可以使用自定义类型来实现。)

A custom type lets you escape to java to define the way in which the field encodes and decodes. To code a custom type you declare your field like this:(自定义类型允许您转义到 java 来定义字段编码和解码的方式。要编写自定义类型，您可以像这样声明您的字段：)

field type="custom:org.apache.poi.hssf.record.LinkedDataFormulaField"
size="var" name="formula of link" description="formula"/>

Where the class name specified after custom: is a class implementing the interface CustomField.(其中custom:后面指定的类名是实现接口CustomField的类。)

You can then implement the encoding yourself.(然后，您可以自己实现编码。)

How it Works(如何运行？)

The record generation works by taking an XML file and styling it using XSLT. Given that XSLT is a little limited in some ways it was necessary to add a little Java code to the mix.(记录生成是通过获取一个 XML 文件并使用 XSLT 对其进行样式设置的方式工作的。鉴于 XSLT 在某些方面受到了一些限制，因此有必要在其中添加一些 Java 代码。)

See record.xsl, record_test.xsl, FieldIterator.java, RecordUtil.java, RecordGenerator.java(参见record.xsl、record_test.xsl、FieldIterator.java、RecordUtil.java、RecordGenerator.java)

There is a corresponding "type" generator for HWPF. See the HWPF documentation for details.(HWPF 有一个相应的“类型”生成器。有关详细信息，请参阅 HWPF 文档。)

Limitations(局限性)

The record generator does not handle all possible record types and goes not intend to perform this function. When dealing with a non-standard record sometimes the cost-benefit of coding the record by hand will be greater than attempting modify the generator. The main point of the record generator is to save time, so keep that in mind.(记录生成器不处理所有可能的记录类型并且不打算执行此功能。在处理非标准记录时，有时手动编码记录的成本效益将大于尝试修改生成器。记录生成器的主要目的是节省时间，所以请记住这一点。)

Currently the the XSL file that generates the record calls out to Java objects. The Java code for the record generation is currently quite messy with minimal comments.(目前，生成记录的 XSL 文件调用 Java 对象。用于记录生成的 Java 代码目前非常混乱，注释很少。)
