HSSF Use Cases(HSSF 用例)

Use Case 1: Read existing HSSF(用例 1：读取现有的 HSSF)

Primary Actor: HSSF client(主要参与者：HSSF 客户)

Scope: HSSF(范围：HSSF)

Level: Summary(级别：总结)

Stakeholders and Interests:(利益相关者和利益：)

  • HSSF client- wants to read content of HSSF file(HSSF 客户端 - 想要读取 HSSF 文件的内容)
  • HSSF - understands HSSF file(HSSF - 理解 HSSF 文件)
  • POIFS - understands underlying POI file system(POIFS - 了解底层 POI 文件系统)

Precondition: None(前提条件：无)

Minimal Guarantee: None(最低保证：无)

Main Success Guarantee:(主要成功保证：)

  1. HSSF client requests HSSF to read a HSSF file, providing an InputStream containing HSSF file in question.(HSSF 客户端请求 HSSF 读取 HSSF 文件，提供包含相关 HSSF 文件的 InputStream。)
  2. HSSF requests POIFS to read the HSSF file, passing the InputStream object to POIFS (POIFS use case 1, read existing file system)(HSSF请求POIFS读取HSSF文件，将InputStream对象传递给POIFS（POIFS用例1，读取现有文件系统）)
  3. HSSF reads the "Workbook" file (use case 4, read workbook entry)(HSSF 读取“工作簿”文件（用例 4，读取工作簿条目）)

Extensions:(扩展：)

2a. Exceptions thrown by POIFS will be passed on to the HSSF client.(2a. POIFS 抛出的异常将传递给 HSSF 客户端。)

Use Case 2: Write HSSF file(用例 2：写入 HSSF 文件)

Primary Actor: HSSF client(主要参与者：HSSF 客户)

Scope: HSSF(范围：HSSF)

Level: Summary(级别：总结)

Stakeholders and Interests:(利益相关者和利益：)

  • HSSF client- wants to write file out.(HSSF 客户端 - 想要写出文件。)
  • HSSF - knows how to write file out.(HSSF - 知道如何写出文件。)
  • POIFS - knows how to write file system out.(POIFS - 知道如何写出文件系统。)

Precondition:(前提：)

  • File has been read (use case 1, read existing HSSF file) and subsequently modified or file has been created (use case 3, create HSSF file)(已读取文件（用例 1，读取现有 HSSF 文件）并随后修改或创建文件（用例 3，创建 HSSF 文件）)

Minimal Guarantee: None(最低保证：无)

Main Success Guarantee:(主要成功保证：)

  1. HSSF client provides an OutputStream to write the file to.(HSSF 客户端提供了一个 OutputStream 来写入文件。)
  2. HSSF writes the "Workbook" to its associated POIFS file system (use case 5, write workbook entry)(HSSF 将“工作簿”写入其关联的 POIFS 文件系统（用例 5，写入工作簿条目）)
  3. HSSF requests POIFS to write its file system out, using the OutputStream obtained from the HSSF client (POIFS use case 2, write file system).(HSSF 使用从 HSSF 客户端获得的 OutputStream 请求 POIFS 将其文件系统写出（POIFS 用例 2，写入文件系统）。)

Extensions:(扩展：)

3a. Exceptions from POIFS are passed to the HSSF client.(3a。 POIFS 的异常被传递给 HSSF 客户端。)

Use Case 3:Create HSSF file(用例 3：创建 HSSF 文件)

Primary Actor: HSSF client(主要参与者：HSSF 客户)

Scope: HSSF(范围：HSSF)

Level: Summary(级别：总结)

Stakeholders and Interests:(利益相关者和利益：)

  • HSSF client- wants to create a new file.(HSSF 客户端 - 想要创建一个新文件。)
  • HSSF - knows how to create a new file.(HSSF - 知道如何创建一个新文件。)
  • POIFS - knows how to create a new file system.(POIFS - 知道如何创建一个新的文件系统。)

Precondition:(前提：)

Minimal Guarantee: None(最低保证：无)

Main Success Guarantee:(主要成功保证：)

  1. HSSF requests POIFS to create a new file system (POIFS use case 3, create new file system)(HSSF请求POIFS创建新文件系统（POIFS用例3，创建新文件系统）)

Extensions: None(扩展：无)

Use Case 4: Read workbook entry(用例 4：读取工作簿条目)

Primary Actor: HSSF(主要演员：HSSF)

Scope: HSSF(范围：HSSF)

Level: Summary(级别：总结)

Stakeholders and Interests:(利益相关者和利益：)

  • HSSF - knows how to read the workbook entry(HSSF - 知道如何阅读工作簿条目)
  • POIFS - knows how to manage the file system.(POIFS - 知道如何管理文件系统。)

Precondition:(前提：)

  • The file system has been read (use case 1, read existing HSSF file) or has been created and written to (use case 3, create HSSF file system; use case 5, write workbook entry).(文件系统已被读取（用例 1，读取现有 HSSF 文件）或已创建并写入（用例 3，创建 HSSF 文件系统；用例 5，写入工作簿条目）。)

Minimal Guarantee: None(最低保证：无)

Main Success Guarantee:(主要成功保证：)

  1. HSSF requests POIFS for the "Workbook" file(HSSF 为“工作簿”文件请求 POIFS)
  2. POIFS returns an InputStream for the file.(POIFS 返回文件的 InputStream。)
  3. HSSF reads from the InputStream provided by POIFS(HSSF 从 POIFS 提供的 InputStream 中读取)
  4. HSSF closes the InputStream provided by POIFS(HSSF 关闭 POIFS 提供的 InputStream)

Extensions:(扩展：)

3a. Exceptions thrown by POIFS will be passed on(3a。 POIFS 抛出的异常将被传递)

Use Case 5: Write workbook entry(用例 5：编写工作簿条目)

Primary Actor: HSSF(主要演员：HSSF)

Scope: HSSF(范围：HSSF)

Level: Summary(级别：总结)

Stakeholders and Interests:(利益相关者和利益：)

  • HSSF - knows how to manage the write the workbook entry.(HSSF - 知道如何管理写入工作簿条目。)
  • POIFS - knows how to manage the file system.(POIFS - 知道如何管理文件系统。)

Precondition:(前提：)

  • Either an existing HSSF file has been read (use case 1, read existing HSSF file) or an HSSF file has been created (use case 3, create HSSF file).(已读取现有 HSSF 文件（用例 1，读取现有 HSSF 文件）或已创建 HSSF 文件（用例 3，创建 HSSF 文件）。)

Minimal Guarantee: None(最低保证：无)

Main Success Guarantee:(主要成功保证：)

  1. HSSF checks the POIFS file system directory for the "Workbook" file (POIFS use case 8, read file system directory)(HSSF 检查 POIFS 文件系统目录中的“工作簿”文件（POIFS 用例 8，读取文件系统目录）)
  2. If "Workbook" is in the directory, HSSF requests POIFS to replace it with the new workbook entry (POIFS use case 4, replace file in file system). Otherwise, HSSF requests POIFS to write the new workbook file, with the name "Workbook" (POIFS use case 6, write new file to file system)(如果“工作簿”在目录中，HSSF 请求 POIFS 将其替换为新的工作簿条目（POIFS 用例 4，替换文件系统中的文件）。否则，HSSF 请求 POIFS 写入新的工作簿文件，名称为“Workbook”（POIFS 用例 6，将新文件写入文件系统）)

Extensions:None(扩展：无)
