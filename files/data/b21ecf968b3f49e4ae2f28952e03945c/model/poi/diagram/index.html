Overview(${t_81ad79971f1f4ae28cead0db32d6b84f})

HDGF is the POI Project's pure Java implementation of the Visio binary (VSD) file format. XDGF is the POI Project's pure Java implementation of the Visio XML (VSDX) file format.(${t_0896ad8af227480c8daf3ca2fa07587a})

Currently, HDGF provides a low-level, read-only api for accessing Visio documents. It also provides a way to extract the textual content from a file.(${t_caabb33814294a71a6a84cd4a3ad9367})

At this time, there is no usermodel api or similar, only low level access to the streams, chunks and chunk commands. Users are advised to check the unit tests to see how everything works. They are also well advised to read the documentation supplied with vsdump to get a feel for how Visio files are structured.(${t_c62430815b024f6986fb234324382cc5})

To get a feel for the contents of a file, and to track down where data of interest is stored, HDGF comes with VSDDumper to print out the contents of the file. Users should also make use of vsdump to probe the structure of files.(${t_1c6f68da35ef47e9937034825915f800})

Steps required for write support(${t_09eca5ba89bf4e8dad249689ea296ae5})

Currently, HDGF is only able to read visio files, it is not able to write them back out again. We believe the following are the steps that would need to be taken to implement it.(${t_8c4175b1f7af4ad98db0fabc994f4626})

  1. Re-write the decompression support in LZW4HDGF as HDGFLZW, which will be much better documented, and also under the ASL. Completed October 2007(${t_7e034e80f88242268af1fc77e05d0c79})
  2. Add compression support to HDGFLZW. In progress - works for small streams but encoding goes wrong on larger ones(${t_b80316fea1cd443a8c652488e79d52cd})
  3. Have HDGF just write back the raw bytes it read in, and have a test to ensure the file is un-changed.(${t_98fc9a06a2d64a879f90f687dce9dff1})
  4. Have HDGF generate the bytes to write out from the Stream stores, using the compressed data as appropriate, without re-compressing. Plus test to ensure file is un-changed.(${t_8db584f9a992476b888f3a56a4afb7ff})
  5. Have HDGF generate the bytes to write out from the Stream stores, re-compressing any streams that were decompressed. Plus test to ensure file is un-changed.(${t_f97d646393eb41238d56434ebef8ff65})
  6. Have HDGF re-generate the offsets in pointers for the locations of the streams. Plus test to ensure file is un-changed.(${t_9045fc2305a7417899388b4b0ff7c595})
  7. Have HDGF re-generate the bytes for all the chunks, from the chunk commands. Tests to ensure the chunks are serialized properly, and then that the file is un-changed(${t_18e6507b6bcd415486c41607762ce4cc})
  8. Alter the data of one command, but keep it the same length, and check visio can open the file when written out.(${t_3e589a6a56584dddae026c9593f22023})
  9. Alter the data of one command, to a new length, and check that visio can open the file when written out.(${t_7102bc7e1d564495be2eb37c08573127})
