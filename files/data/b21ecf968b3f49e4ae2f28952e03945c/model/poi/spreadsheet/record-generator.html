How to Use the Record Generator(${t_72118ba0e9b84dc8bd2362e0a96fe49b})

History(${t_23d87a60b5f3456da4d94127e1423b72})

The record generator was born from frustration with translating the Excel records to Java classes. Doing this manually is a time consuming process. It's also very easy to make mistakes.(${t_f6c30423f1f54c64a3ee8c82d06ec65e})

A utility was needed to take the definition of what a record looked like and do all the boring and repetitive work.(${t_aa4ce49228ae404cbb51d317a487152c})

Capabilities(${t_7fff282a8f604742b324f1fa77100538})

The record generator takes XML as input and produces the following output:(${t_962cba1c7d55497e99a5a8691ca5305d})

  • A Java file capable of decoding and encoding the record.(${t_e141e85f66bf4f1fbd7248f06a8718f8})
  • A test class that provides a fill-in-the-blanks implementation of a test case for ensuring the record operates as designed.(${t_feaa0809d0da486baad7d4298915ccd3})

Usage(${t_204acd0d8ba54a358846a283e630b953})

The record generator is invoked as an Ant target (generate-records). It goes through looking for all files in src/records/definitions ending with _record.xml. It then creates two files; the Java record definition and the Java test case template.(${t_518f8ee66fcc4745ae4784ed2ccebbab})

The records themselves have the following general layout:(${t_a318d52569ef493e9ae29c2b47e4eb56})

record id="0x1032" name="Frame" package="org.apache.poi.hssf.record"
excel-record-id="FRAME">
<description>The frame record indicates whether there is a border
around the displayed text of a chart.</description>
<author>Glen Stampoultzis (glens at apache.org)</author>
<fields>
<field type="int" size="2" name="border type">
<const name="regular" value="0" description="regular rectangle or no border"/>
<const name="shadow" value="1" description="rectangle with shadow"/>
</field>
<field type="int" size="2" name="options">
<bit number="0" name="auto size"
description="excel calculates the size automatically if true"/>
<bit number="1" name="auto position"
description="excel calculates the position automatically"/>
</field>
</fields>
/record>

The Java records are regenerated each time the record generator is run, however the test stubs are only created if the test stub does not already exist. What this means is that you may change test stubs but not the generated records.(${t_a946d425dfcb4b23b7578f5f1d38be63})

Custom Field Types(${t_7512d0b118ec4d3586e0bb01bdef6138})

Occasionally the builtin types are not enough. More control over the encoding and decoding of the streams is required. This can be achieved using a custom type.(${t_481c30c2733f4da594edae6a9b9ff9a6})

A custom type lets you escape to java to define the way in which the field encodes and decodes. To code a custom type you declare your field like this:(${t_c3b6b277c83d45339c368dfe4a310211})

field type="custom:org.apache.poi.hssf.record.LinkedDataFormulaField"
size="var" name="formula of link" description="formula"/>

Where the class name specified after custom: is a class implementing the interface CustomField.(${t_47b71bc86acf48bb8c88ee81c12546b0})

You can then implement the encoding yourself.(${t_448d952a0dbc43548484ebdfa56d82f6})

How it Works(${t_6d6d495b492941b7a644f6aaa50816d9})

The record generation works by taking an XML file and styling it using XSLT. Given that XSLT is a little limited in some ways it was necessary to add a little Java code to the mix.(${t_839ddd25b30c4615835d7dedf8f664da})

See record.xsl, record_test.xsl, FieldIterator.java, RecordUtil.java, RecordGenerator.java(${t_8761df77366d4b1cbc6b5f7891631a52})

There is a corresponding "type" generator for HWPF. See the HWPF documentation for details.(${t_25f39e7aa54b4a87b495e03c89fb776f})

Limitations(${t_fa74db08784646bbac4604d1d3d16829})

The record generator does not handle all possible record types and goes not intend to perform this function. When dealing with a non-standard record sometimes the cost-benefit of coding the record by hand will be greater than attempting modify the generator. The main point of the record generator is to save time, so keep that in mind.(${t_cfa6551257dd4cd5803eb65419ec2c4e})

Currently the the XSL file that generates the record calls out to Java objects. The Java code for the record generation is currently quite messy with minimal comments.(${t_a9bbdce9ee3643e39ffd110e11bebcba})
